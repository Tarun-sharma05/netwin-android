🔍 Explanation by Layer
1. Data Layer
Model: All data classes like User, Match, WalletTransaction, LeaderboardEntry, etc.

Repository: Provides functions like getMatches(), joinMatch(), addMoney(), which interact with Firebase.

Source:

firebase: Handles Firestore reads/writes, Auth (OTP or Google), and Firebase Storage (profile/KYC uploads).

local: Optional Room DB for caching or offline support.

2. Domain Layer
UseCases: Clean separation of logic. Example:

kotlin
Copy
Edit
class JoinMatchUseCase(private val repo: MatchRepository) {
    suspend fun invoke(matchId: String, userId: String) = repo.joinMatch(matchId, userId)
}
Utils: Constants (e.g., Firestore collection names), date/time helpers, validations (e.g., isValidID()).

3. Presentation Layer
Built with Jetpack Compose

Follows MVVM pattern (each screen has ViewModel)

Examples:
auth: login, register, password recovery, KYC upload

home: shows list of tournaments, filters (solo/duo/squad), join options

match: displays match info, room ID/password 15 min before

wallet: add money (Razorpay/UPI), withdraw, view balance

admin: only accessible by admin login; create/edit match, enter results
